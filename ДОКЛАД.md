
# Доклад о разработке платформы мониторинга и автоматизации образовательных процессов

## Обзор проекта

Разработана комплексная веб-платформа для управления и мониторинга образовательных процессов. Платформа предназначена для автоматизации работы образовательных учреждений и отслеживания прогресса студентов преподавателями, а не для самостоятельного обучения.

**Стек технологий:**
- **Backend:** FastAPI, SQLModel, SQLAlchemy, Alembic (миграции БД), PostgreSQL
- **Frontend:** React, TypeScript, TanStack Router, TanStack React Query, Tailwind CSS
- **DevOps:** Docker, Docker Compose
- **Аутентификация:** JWT токены, роли (admin, teacher, student)

## Реализованные функции

### 1. Управление образовательными программами

#### Основной функционал:
- ✅ **CRUD операции** - создание, чтение, обновление, удаление программ
- ✅ **Статусы программ** - Черновик, На проверке, Одобрена, Отклонена
- ✅ **Поиск и фильтрация** - быстрый поиск по названию, фильтрация по статусу
- ✅ **Сортировка** - по названию, дате создания, статусу

#### Интерфейс:
- Карточная сетка программ (grid layout)

  - **Модули** - учебные элементы программы
  - **Группы** - группы студентов, обучающихся по программе
  - **Преподаватели** - учителя, ведущие программу

#### Модели данных:
```
Program
├── id (UUID)
├── title (string)
├── description (text)
├── status (draft|on_review|approved|rejected)
├── created_at (timestamp)
└── Relationship: modules, groups
```

### 5. Отслеживание прогресса студентов

```

├── module_id (UUID, FK)
```

- 🟢 **Завершён** - модуль успешно пройден

#### Функционал:

- ✅ **Управление преподавателем** - только учитель может обновлять прогресс (чекбоксы)
- ✅ **Запрет самостоятельного отчёта** - студенты видят свой прогресс (read-only), но не могут его менять
- ✅ **Траектория обучения** - студент видит свой прогресс по программам и модулям

## Интеграция компонентов и архитектура

- `/` - главная страница
- `/login` - страница входа
- `/programs` - список программ

## Технические вызовы и решения

### 1. SQLAlchemy Enum не конвертировалась в PostgreSQL
**Проблема:** Python Enum type не корректно сохранялся в БД, всегда показывал enum name вместо value.
**Решение:** Переконвертирован на plain string type (varchar), создана миграция для alter column.

### 2. Circular imports в CRUD модулях
**Проблема:** Импорт CurrentUser в crud_program приводил к циклической зависимости.
**Решение:** Удалены неиспользуемые импорты, сохранены необходимые.

### 3. TypeScript типизация React Query
**Проблема:** Ошибки типов при использовании placeholderData.
**Решение:** Добавлены явные type annotations для результатов useQuery.

### 4. TanStack Router динамическая маршрутизация
**Проблема:** Файл programs.$programId вместо programs_.$programId создавал вложенные маршруты.
**Решение:** Использована правильная нотация programs_.$programId для параметров на конце.

### 5. Маршруты не регистрировались в routeTree
**Проблема:** Новые маршруты не появлялись в автогенерируемом routeTree.gen.ts.
**Решение:** Вручную добавлены регистрации в FileRoutesByPath, FileRoutesByTo, FileRoutesById интерфейсы.

### 6. Траектория показывала ID программ вместо названий
**Проблема:** enrollment.group объект не был загружен в API response.
**Решение:** Реализована логика отдельной загрузки групп и программ с созданием lookupMap.

### 7. Модули не отображались в траектории
**Проблема:** useQuery для модулей не вызывалась с параметром program_id.
**Решение:** Добавлена явная передача program_id в queryFn.
